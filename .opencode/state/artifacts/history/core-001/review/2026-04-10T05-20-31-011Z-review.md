## Reviewer Notes

The plan at `.opencode/state/plans/core-001-planning-plan.md` is comprehensive and technically sound. All 5 acceptance criteria are explicitly addressed, the Godot 4.6 specifics are correct (Area2D `monitoring` property, delta-based timers, `has_animation()` guard pattern), the SFX graceful fallback is well-designed, and the defensive coding for the not-yet-present enemy system is appropriate.

The plan correctly scopes CORE-001 to: attack input triggering, HitboxArea activation window, cooldown enforcement, animation trigger, and SFX placeholder. The actual signal wiring and damage-dealing belongs to CORE-005 (Collision/damage system), which is the correct dependency boundary.

The plan's `_input()` code adds `elif event is InputEventScreenTouch` and `elif event is InputEventScreenDrag` at the top level of the function (after the existing joystick `if`/`elif` chain). Since the new code's touch condition is `event.position.x >= screen_size.x / 2` and the existing code's touch condition is `event.position.x < screen_size.x / 2`, they are mutually exclusive — no conflict at runtime. The new drag handler correctly uses the already-stored `_attack_touch_start` to determine if the original press was on the right side.

Pitfalls documented in Section 11 are accurate, particularly the Area2D `monitoring` reset pattern and the animation force-restart using `play("attack", true)`.

## Technical Soundness

**Godot 4.6 Area2D:** Correct. The plan uses `monitoring = true` to enable HitboxArea detection during the attack window, and correctly notes that `monitorable` is not needed for the player's HitboxArea (only for areas that need to be detected by others).

**Delta-based timers:** Correct. Using `_attack_window` and `_attack_cooldown` floats decremented in `_physics_process()` is the standard Godot pattern that avoids extra Timer nodes.

**Animation guard pattern:** Correct. `has_animation("attack")` is the proper Godot 4.x check before calling `play()` on an AnimatedSprite2D — prevents runtime errors when the animation doesn't exist yet.

**SFX graceful fallback:** Correct. `ResourceLoader.exists()` before playback is the standard Godot pattern for optional audio assets.

**Held attack handling:** Correct. The `_attack_held` flag approach properly distinguishes tap-attack from movement drags on the right side. The `ATTACK_HOLD_THRESHOLD` (8.0px drag) is reasonable.

**Defensive signal wiring:** Correct. The plan defers `body_entered` signal connection to CORE-005 and uses `has_method("take_damage")` as a defensive interface check — appropriate for cross-ticket dependency safety.

**No structural scene changes needed:** Correct. The player.tscn already has HitboxArea, HitboxShape, HurtboxArea, and HurtboxShape as children of the Player CharacterBody2D. No new nodes required.
