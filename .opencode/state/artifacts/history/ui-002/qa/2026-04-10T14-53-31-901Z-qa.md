## Detailed Findings

### AC4 — FAIL (Blocking)

**File:** `scenes/ui/game_over.gd` line 11
```gdscript
WaveLabel.text = "WAVE: %d" % GameManager.wave
```

**Problem:** `GameManager.wave` is the *current* wave counter. When `GameManager.reset()` is called (lines 14, 18), `wave` is reset to `1` (game_manager.gd line 21). The highest wave reached is stored in `max_wave` (game_manager.gd lines 5, 16-17), which persists across reset calls.

The acceptance criterion explicitly reads: **"highest wave reached"**. The current implementation always shows `WAVE: 1` on the game over screen because `reset()` zeroed `wave` before the scene displays.

**Fix:** Change line 11 to:
```gdscript
WaveLabel.text = "WAVE: %d" % GameManager.max_wave
```

### AC5 — Minor Note (Non-Blocking)

game_over.tscn uses plain `Button` nodes (lines 60, 64) rather than `TextureButton` with sourced UI sprite assets. This is acceptable since the acceptance criterion does not mandate active application of the sprites, and the font + sprite assets are confirmed present in the project. This is a design consistency observation, not a QA blocker.
