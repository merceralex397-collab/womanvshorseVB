# Implementation: UI-002 Game Over Screen

## Stage
implementation

## Ticket
UI-002: Game over screen

## Summary
Created game over screen with score/wave display, retry and menu buttons, and GameManager autoload for state persistence.

## Files Created/Modified

### 1. Created `scripts/autoloads/game_manager.gd`
GameManager singleton for persisting score, wave, and max_wave across scenes.
- `score: int` — current score
- `wave: int` — current wave number
- `max_wave: int` — highest wave reached (persists across runs)
- `add_score(points)` — adds points to score
- `set_wave(w)` — updates wave, tracks max_wave
- `reset()` — resets score/wave but preserves max_wave

### 2. Created `scenes/ui/game_over.gd`
Script for game over screen:
- Reads `GameManager.score` and `GameManager.wave` on `_ready()`
- `_on_retry_pressed()` — resets GameManager, loads main.tscn
- `_on_menu_pressed()` — resets GameManager, loads title_screen.tscn

### 3. Rewrote `scenes/ui/game_over.tscn`
Full scene replacing the minimal stub:
- Control root with FULL_RECT anchors
- ColorRect dark overlay background (Color(0.1, 0.0, 0.0, 0.85))
- CenterContainer → VBoxContainer layout with:
  - "GAME OVER" label (red font color)
  - "SCORE:" dynamic label
  - "WAVE:" dynamic label
  - Spacer node
  - Retry Button
  - Menu Button
- Signal connections for button pressed events

### 4. Updated `project.godot`
Added `[autoload]` section:
```
[autoload]
GameManager="*res://scripts/autoloads/game_manager.gd"
```
