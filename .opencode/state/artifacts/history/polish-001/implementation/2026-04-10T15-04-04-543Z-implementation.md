# Implementation Artifact — POLISH-001: Particle Effects

## Ticket
- **ID**: POLISH-001
- **Title**: Particle effects
- **Stage**: implementation
- **Wave**: 4
- **Lane**: polish

## Summary
Implemented GPUParticles2D particle effects for player attack arc, enemy hit burst, enemy death explosion, player damage burst, and wave clear celebration. All particles use built-in particle textures (no external assets required).

## Changes Made

### 1. Created `scripts/particles/particle_effect_helper.gd`

A helper script providing `create_particles()` and `spawn_particles()` functions for GPUParticles2D setup. The helper:
- Creates `GPUParticles2D` nodes with `ParticleProcessMaterial`
- Sets `one_shot = true` for burst-style effects
- Auto-cleanups particles via `finished` signal connection

### 2. Modified `scenes/player/player.gd`

**In `_try_attack()` — attack arc particles (AC1):**
- Spawns 20 golden yellow particles with 0.3s lifetime, 0.8 explosiveness
- Direction: horizontal (Vector3(1, 0, 0)), spread: 80.0°
- Velocity: 150-300, scale: 3-6, gravity: 200-400

**In `take_damage()` when health > 0 — player damage burst (AC4):**
- Spawns 12 red particles with 0.25s lifetime, 0.9 explosiveness
- Direction: upward (Vector3(0, -1, 0)), spread: 180.0°
- Velocity: 80-150, scale: 2-4, gravity: 200-400

### 3. Modified `scenes/enemies/horse_base.gd`

**In `take_damage()` when health > 0 — hit burst (AC2):**
- Spawns 12 orange-red particles with 0.25s lifetime, 0.9 explosiveness
- Same parameters as player damage burst
- Plays after hurt animation trigger

**In `die()` — death explosion (AC3):**
- Spawns 40 brown-orange particles with 0.6s lifetime, 0.7 explosiveness
- Velocity: 100-250, scale: 4-8, gravity: 200-500
- Emits `died` signal before particles (preserving score handling)
- `queue_free()` still occurs after animation finishes

### 4. Modified `scripts/wave_spawner.gd`

**In `_on_wave_cleared()` — wave clear celebration (AC5):**
- Connected `self.wave_cleared.connect(_on_wave_cleared)` in `_ready()`
- Spawns 60 gold particles with 1.0s lifetime, 0.5 explosiveness
- Position: arena center (640, 360)
- Velocity: 150-350, scale: 4-8, gravity: 100-300

### 5. Added autoload to `project.godot`

```
[autoload]
ParticleEffectHelper="*res://scripts/particles/particle_effect_helper.gd"
```

## Risks Mitigated

- **Risk**: `die()` in horse_base.gd did `queue_free()` directly without waiting for particles
  - **Mitigation**: `died.emit()` still fires first (preserving score/signal chain); particles play while death animation runs; `queue_free()` happens after animation via existing `await sprite.animation_finished`
- **Risk**: Wave spawner `wave_cleared` signal already connected elsewhere
  - **Mitigation**: Signal connection is additive (multiple receivers can connect); celebration is supplementary
