## Code Correctness Assessment

- **Player script (`player.gd`)**: Attack arc and damage burst helper methods (`_create_attack_arc_material`, `_create_damage_burst_material`) are well-specified with correct `ParticleProcessMaterial` properties. Spawn in correct lifecycle positions (`_try_attack()` after hitbox activation, `_take_damage()` after modulate flash).

- **Horse base script (`horse_base.gd`)**: `die()` replacement correctly preserves `died.emit()` before the explosion flow, ensuring HUD score handlers fire before `queue_free()`. The `finished` callback chains `explosion.queue_free()` → `queue_free()` which is safe.

- **Wave spawner (`wave_spawner.gd`)**: Celebration burst is additive (does not interfere with existing signal handlers), positioned at arena center `Vector2(540, 300)` for a 1080×600 arena. `wave_cleared.connect(_on_wave_cleared)` added in `_ready()`.

- **No new scenes or assets required** — matches the no-placeholder-art constraint from canonical brief.

## Godot 4.6 API Correctness

- `GPUParticles2D` with `ParticleProcessMaterial` is the correct Godot 4.6 pattern for programmatic 2D particles.
- `Curve.create_from_csv("0,1|1,0")` is a valid inline curve construction for alpha fade-out.
- `finished.connect(lambda)` signal connection style is valid GDScript.
- All node additions use `add_child()` and self-cleanup via `finished` signal — correct resource management.
