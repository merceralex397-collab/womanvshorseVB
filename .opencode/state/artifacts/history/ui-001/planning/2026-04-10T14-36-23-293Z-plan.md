# Plan: UI-001 Title Screen

## 1. Scope

Create the title screen scene (`scenes/ui/title_screen.tscn`) and its script (`scenes/ui/title_screen.gd`) that serves as the game's entry point, with a styled title, a start button, and a credits button. Update `project.godot` to set the title screen as the main scene.

## 2. Files / Systems Affected

| File | Change |
|---|---|
| `scenes/ui/title_screen.tscn` | New — title screen scene |
| `scenes/ui/title_screen.gd` | New — button signal handlers |
| `project.godot` | Modify — change `run/main_scene` to title_screen |

## 3. Implementation Steps

### Step 1 — Create `scenes/ui/title_screen.gd`

```gdscript
extends Control

func _ready() -> void:
    $StartButton.pressed.connect(_on_start_pressed)
    $CreditsButton.pressed.connect(_on_credits_pressed)

func _on_start_pressed() -> void:
    get_tree().change_scene_to_file("res://scenes/main.tscn")

func _on_credits_pressed() -> void:
    # UI-003 creates the credits scene; point to the planned path.
    # Graceful no-op until UI-003 is complete.
    var credits_path = "res://scenes/ui/credits.tscn"
    if ResourceLoader.exists(credits_path):
        get_tree().change_scene_to_file(credits_path)
```

### Step 2 — Create `scenes/ui/title_screen.tscn`

- **Root node:** `Control` (full-screen anchor preset 15, grow both axes)
- **Background:** `TextureRect` stretching to fill screen using a panel background sprite from `assets/sprites/ui/` (e.g. `button_rectangle_gloss.png` as a background panel, or a dedicated panel sprite if one exists — fallback to solid dark fill via `ColorRect` if no suitable panel sprite is identified)
- **Title Label:** `Label` child, centered horizontally, upper third of screen
  - Text: `"WOMAN VS HORSE"`
  - Font: Press Start 2P loaded dynamically from `res://assets/fonts/PressStart2P-Regular.ttf`
  - Font size: 32px or largest available in the font
- **Start Button:** `TextureButton` child, centered, below title
  - Normal texture: `assets/sprites/ui/button_square_gloss.png` (or `button_rectangle_gloss.png`)
  - Hover / pressed textures: use the same sprite with hover_over sound (no audio implemented yet — silent)
  - Label text on button: `"START"`
- **Credits Button:** `TextureButton` child, below Start button
  - Same texture set as Start button
  - Label text on button: `"CREDITS"`

**Scene tree:**
```
title_screen (Control)
├── BackgroundRect (TextureRect or ColorRect)
├── TitleLabel (Label)
├── VBoxContainer (VBoxContainer, centered)
│   ├── StartButton (TextureButton with Label child)
│   └── CreditsButton (TextureButton with Label child)
```

### Step 3 — Update `project.godot`

Change under `[application]`:
```
run/main_scene="res://scenes/ui/title_screen.tscn"
```

## 5. Risks & Assumptions

- **Risk:** `credits.tscn` (UI-003) does not exist yet. The credits button uses `ResourceLoader.exists` guard so it will silently no-op until UI-003 is complete. This is intentional to avoid blocking UI-001 on UI-003.
- **Assumption:** `button_square_gloss.png` or `button_rectangle_gloss.png` from Kenney UI Pack is suitable as a button background. If the sprite orientation is wrong (the Kenney UI buttons are horizontal strips with normal/hover/pressed cells), a `Button` node with `add_theme_stylebox_override` may be needed instead of `TextureButton`. If that is the case, switch to a `Button` node with the appropriate `StyleBoxTexture`.
- **Risk:** Button labels require a child `Label` node (TextureButton does not have built-in text). This is the established pattern already used in the HUD scene structure.
- **Assumption:** The 1280×720 viewport and `canvas_items` stretch mode from `project.godot` means hardcoded pixel sizes in `title_screen.tscn` will scale correctly on device.

## 6. Blocker / Required User Decision

None. All required assets are present (ASSET-004 done). The credits scene path is guarded so UI-001 does not block on UI-003.
