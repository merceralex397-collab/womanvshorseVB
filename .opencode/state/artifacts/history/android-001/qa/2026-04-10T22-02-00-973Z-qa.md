## AC2: android/ directory has required support files

**Verdict:** FAIL

**Evidence:**

```
$ ls android/
scafforge-managed.json
```

```
$ ls -la android/
total 12
drwxr-xr-x 2 pc pc 4096 Apr  9 22:47 .
drwxrwxr-xr-x 1 pc pc 4096 Apr 10 12:07 ..
-rw-r--r-- 1 pc pc  526 Apr  9 22:47 scafforge-managed.json
```

The `android/` directory contains only `scafforge-managed.json` — a scaffold metadata marker file. It does NOT contain:

- `build.gradle` (or `build.gradle.kts`)
- `settings.gradle` (or `settings.gradle.kts`)
- `AndroidManifest.xml`
- Gradle wrapper files (`gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.jar`)

A scafforge-managed.json marker is NOT a Gradle project file. AC2 definitively fails. The android/ directory is empty of Android project surfaces.

```

(command returned no output — JAVA_HOME not set in environment)
```

```
$ printenv ANDROID_HOME
(command returned no output — ANDROID_HOME not set in environment)
```

- JAVA_HOME: **NOT SET** in environment
- ANDROID_HOME: **NOT SET** in environment  
- Debug keystore: **EXISTS** at `/home/pc/.local/share/godot/keystores/debug.keystore` (per export_presets.cfg) but NOT at `~/.android/debug.keystore`

AC3 is partial. Debug keystore exists but environment variables for Java/Android SDK are not exported. Shell export restrictions block the APK production path.

**Concrete blocker:** The `android/` directory lacks Gradle project surfaces (`build.gradle`, `settings.gradle`, `AndroidManifest.xml`, gradle wrapper). APK production is blocked because Godot's Android export requires these Gradle files to be present alongside `android/` for the Gradle build system to function. Shell export restrictions further constrain the APK production path.
